# Per-tile difficulty

Every playable cell has a difficulty rating **1–6** (1 = unmissable landmark,
5 = anonymous corridor, 6 = never served as a target in any mode). Ratings
drive three things at runtime:

- **Pool filtering** — the menu tiers draw round targets from a rating band
  (`Difficulty.min`–`max` in `src/scoring.ts`): Recruit 1–3, Bounty Hunter 2–4,
  Chozo Warrior 3–5. If a band holds fewer cells than a run needs,
  `pickTargets` falls back to the full pool.
- **Score multiplier** — `tileMult(rating)` = ×0.75 at rating 1 up to ×1.25 at
  rating 5; the tile itself carries the reward, there is no per-tier multiplier.
- **Exclusion** — rating 6 (`EXCLUDED_RATING`) marks degenerate screens (flat
  sky, empty water, featureless murk) that are unguessable; they never enter
  the target pool, not even via the fallback.

## Data files

- `public/data/difficulty.<game>.json` — flat `{ "areaId:tileX,tileY": rating }`
  in **tile** coordinates, the same keying as `roomNames.<game>.json`. Loaded by
  `loadGameData` as an overlay; a missing file or missing key means the neutral
  default rating **3** (`DEFAULT_RATING`). Generated by `pipeline/rate_tiles.py`
  and hand-tweakable in the editor.
- `pipeline/room-difficulty.<game>.json` — the hand-curated name signal:
  every room name → base rating 1–5 (1 = Landing Site / boss arenas; 2 =
  distinctive set pieces and item landmarks; 3 = typical rooms and stations;
  4 = generic connectors; 5 = anonymous sand pits / quicksand / speedways).
  Curated against community perspective (wiki.supermetroid.run room list).
  **This is the durable source of truth** — `rate_tiles.py` overwrites
  `difficulty.<game>.json`, so fold lasting hand-tweaks back into this table
  (or accept that they must be re-painted after a regeneration).

## The generator (`pipeline/rate_tiles.py`)

Blends two per-cell signals (needs Pillow + numpy):

1. **Name base** — the cell's room rating from the curated table, joined via
   `roomNames.<game>.json`. Fails loudly if any named room is missing from the
   table or any playable cell is unnamed.
2. **Visual distinctness** — feature vector per tile PNG (16×16 grayscale
   structure + 4×4×4 RGB palette histogram), mean euclidean distance to the
   k=3 nearest other tiles in the game, percentile-ranked. Near-clones rate
   harder, one-of-a-kind screens easier.

`raw = 0.55·base + 0.45·(5 − 4·pct)`, stretched ×1.4 around 3 (averaging two
signals compresses toward the middle), rounded, clamped to 1–5.

**Degenerate screens** are detected by edge energy (mean absolute luminance
gradient) below 4.5 → rating 6. Luminance mean/std were tried first and don't
work: dark caves with real structure score low on both, while featureless murk
gradients score high. The observed gap: featureless tiles ≤ ~4.2, guessable
dark rooms ≥ ~5.3.

Super Metroid distribution (2026-07): 146 / 385 / 431 / 198 / 33 / 25 across
ratings 1–6.

## Editor

The icon editor's **Diff** tool (round header → icons): pick a rating 1–6 in
the toolbar, click playable cells to paint them one at a time. While the tool
is active the map tints every playable cell by rating (green → red, 6 blacked
out; unrated cells show the default's color faintly). **Save to file** writes
`difficulty.<game>.json` through the same dev middleware as the other overlays.
